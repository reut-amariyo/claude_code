# Professional Email Workflow

> A collaborative workflow between the Copywriter and the Gatekeeper for creating a professional email.

---

## Process Overview

```
[You] provide context and purpose
          ↓
[Copywriter] writes first draft
          ↓
[Gatekeeper] reviews and approves / requests changes
          ↓
[Copywriter] revises (if needed)
          ↓
[Gatekeeper] final approval
          ↓
[You] send
```

---

## Step 1: Preparation

### Before Starting

Both agents must read:
- `C-core/project-brief.md` - Who you are and what you do
- `C-core/voice-dna.md` - How your brand sounds
- `C-core/icp-profile.md` - Who your target audience is
- `M-memory/learning-log.md` - What we learned from previous rounds

### Project Setup

Create a new folder in `O-output/` with a clear name:
```
O-output/
└── 01-email-[purpose]/
    ├── copywriter-draft.md    ← Copywriter's version
    ├── gatekeeper-review.md   ← Gatekeeper's review
    └── final-email.md         ← Final version, ready to send
```

---

## Step 2: Copywriter Creates a Draft

### What the Copywriter Does

1. **Reads C-core documents** - Understands the voice and audience
2. **Reads `T-tools/01-skills/professional-email-skill/`** - Learns the correct format
3. **Writes first draft** - Matching the right email type

### File Format: `copywriter-draft.md`

```markdown
# Email: [Purpose / Recipient]

**Date:** [Date]
**Version:** v1
**Email type:** [outreach / follow-up / update / proposal / introduction]

---

## Subject Line

[Subject line here]

---

## The Email

[Email text goes here]

---

## Copywriter Notes

### Choices I Made
- [Why I chose this subject line]
- [Why this tone]
- [What the main ask is]

### Alternatives I Considered
- [Different subject line version]
- [Different opening]

### Questions for the Gatekeeper
- [Is the tone right?]
- [Is the ask clear?]
```

---

## Step 3: Gatekeeper Reviews

### What the Gatekeeper Does

1. **First read** - Would I open this email?
2. **Check against standards** - Voice DNA, clarity, clear ask
3. **Decision** - Approved / Revisions Needed / Escalate

### File Format: `gatekeeper-review.md`

```markdown
# Gatekeeper Review: [Purpose / Recipient]

**Date:** [Date]
**Version reviewed:** v1

---

## Status: [APPROVED / REVISIONS NEEDED / ESCALATE]

---

## What Works Well
- [Strength 1]
- [Strength 2]

## What Needs Improvement
1. **[Issue 1]** - [How to fix]
2. **[Issue 2]** - [How to fix]

---

## Next Step
[What the Copywriter should do now]
```

---

## Step 4: Revision (If Needed)

If the Gatekeeper requested revisions:

1. **Copywriter reads the notes** - Understands what to fix
2. **Updates `copywriter-draft.md`** - Bumps version to `v2`
3. **Gatekeeper reviews again** - Updates `gatekeeper-review.md`

---

## Step 5: Final Version

When the Gatekeeper approves:

### File Format: `final-email.md`

```markdown
# Final Email: [Purpose / Recipient]

**Status:** Approved for sending
**Approval date:** [Date]

---

**Subject:** [Subject line]

---

[Final email text - clean, no notes, ready to copy-paste]

---

*Created by: Copywriter Agent*
*Reviewed by: Gatekeeper Agent*
```

---

## Step 6: Update Learning Log

After the process is complete, the Gatekeeper updates `M-memory/learning-log.md`:

```markdown
## [Date] - Email: [Purpose / Recipient]

### What Worked
- [Insight]

### What We Learned
- [Lesson for next time]
```

## Quick Checklist

### Before Starting
- [ ] New folder created in O-output
- [ ] Both agents read C-core documents
- [ ] Recipient, purpose, and ask are clear

### Copywriter
- [ ] Read voice DNA and ICP
- [ ] Read the Email skill
- [ ] Wrote draft with notes
- [ ] Saved to `copywriter-draft.md`

### Gatekeeper
- [ ] Subject line makes you want to open it?
- [ ] First sentence about the recipient, not about us?
- [ ] One clear ask?
- [ ] Under 200 words?
- [ ] Saved to `gatekeeper-review.md`

### Wrap-Up
- [ ] Final version saved to `final-email.md`
- [ ] Learning log updated
- [ ] Ready to send

---

## Tips for Success

1. **Short = good** - Long emails don't get read
2. **First sentence = about them** - Don't open by talking about yourself
3. **One ask** - Don't request 3 things in one email
4. **Sound like you** - If you wouldn't say it in a conversation, don't write it
